# HDN vs grsecurity Functional Equivalence Matrix

This document compares the current HDN Linux 7.0.12 hardening patch against
the grsecurity/PaX feature surface visible in the local `grsec-6.6.102.patch`.
It is a functional comparison only. HDN should keep using its own policy model,
call sites, tests, and implementation structure.

## Status Key

- Covered: HDN has a kernel mechanism and a smoke proof for the security effect.
- Partial: HDN covers the main attack surface, but not every grsecurity mode,
  compatibility option, or edge case.
- Baseline: HDN relies on upstream Linux Kconfig/runtime mitigations selected
  by `SECURITY_HARDENING_EXPLOIT_MITIGATIONS`.
- Gap: not implemented yet, or only present as a design target.
- Out of scope now: not required for the first daily-driver security baseline.
